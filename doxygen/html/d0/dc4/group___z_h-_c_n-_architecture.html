Introduction to OTA Hotel Management Architecture

架构

OTA酒店项目是一个简单的基于网页的酒店库存管理系统
此软件为开源免费为私人和商业用途使用

OTA酒店项目是受通用公共许可证保护的

OTA酒店项目是从原来的开源计划HotelMIS进行的重新开发，但原始的源代码很少被用到。
OTA酒店项目创建来作为E - Novate私人有限公司E - Bridge软件的测试平台，该软件采用开放式旅游联盟消息规范。
OTA酒店项目使用到E - Novate私人有限公司的一个受商业保护的程序库
这个程序库供任何酒店，软件开发公司免费使用，软件的最终用户使用E - Novate私人有限公司的E - Bridge服务来提供预订的在线接口。

E - Novate私人有限公司的E - Bridge的软件解决方案，是一种低成本的连接和存在的解决方案，允许企业对企业（B2B）和企业对客户（B2C）应用的通用互动。
对于有关的E - Bridge的产品的详细信息，请联系E - Novate，sales@e-novate.asia

对OTA酒店用户的免费在线酒店预订解决方案

OTA酒店软件的用户资格免费注册并免费访问直接连接到OTA酒店软件的在线预订网站。
这是一个专门设计的低成本的新服务，使各级酒店在线和在移动市场竞争。
This product is due for full release before end 2011

此产品将于2011年年底前完全发布

代码布局

代码分为功能和页面。
其主要的功能文件是：:

  • configuration.inc.php - 配置定义
  • hotel_defs.php - 常数
  • login_check.inc.php - 简单的登入核查函数(来自原来的hotelmis)
  • lookupfunctions.php
  • queryfunctions.php - 数据库查询格式函数
  • functions.php - 初级运营函数列表
  • webtoPDF.php - 转换和推出html到PDF的函数

所有其他php文件应该直接输出html和显示酒店网页

语言-翻译

在每个php页面的开始都有3行代码 include_once("lang/lang_en.php");
$lang = get_language();
load_language($lang);

这3行代码，确保至少有英文语言包是加载到每个网页。
这些应在每个文件的开头，以确保正确的语言可移植性。

这3行代码，确保至少有英文语言包是加载到每个网页。. 这些应在每个文件的开头，以确保正确的语言可移植性。.

lang - 国际语言编码
Description - 英语描述
LocalDescription - UTF-8 编码语言版本
active - 0表示可用但不可选，1表示活跃且语言文件存在

INSERT INTO `languages`(`lang`,`Description`,`LocalDescription`,`active`) VALUES ( 'my','Myanmar (Burmese)','Myanmar (Burmese)','0');

要创建一个新的的语言文件，复制lang/ lang_en.php成为lang/ lang_.php，然后编辑该文件
更新数据库,并设置工作状态为1，然后就可以从酒店设置页面设置语言
