Invoices or bills are normally generated at time of reservation confirmation or guest registration at time of check in.
Invoice numbers are generated automatically by incrementing the invoice number in the document setup.
Voucher numbers are collected from the reservation.
Guest name is passed via the guest id from either the reservation or guest registration.
Room number is collected from the guest registration.
Date in is the check in date, this will help with automatic calculation of room charges.

Adhoc bills/invoices may be created for guests that are not checked in. Should this be the case for guests not staying in the hotel, it may be necessary to create an anonymous guest profile, this will save creating profiles for one time sales.

Fees and Charges
To add a new charge, select the date and time the charge was generated by selecting the calendar icon. Then select the item to be charged from the pull down list. Add the standard amount, service charge and tax.
If a special promotion rate is being applied, this will be automatically added to the rates code pull down. Select the applicable rate.
Charge items can not be deleted once they have been added, however they can be voided from the invoice. The garbage bin on the left hand column will void the invoice line item if selected. There is no "unvoid" option, so if an item is voided by mistake, it must be re-added manually.
If the charged amount, service fee and tax are left blank, these will automatically be populated from the standard rates using the promotional rate codes being applied.
If there is no promotional rate code selected the standard amount, service fee and tax will be directly copied across.
To override the calculated amount, simply fill in the charged amount, service fee and tax.

Receiving Payments
Receiving payments is very similar to adding charges, simply add in the date of payment.
Once the Balance Due is 0, pressing the Update button again will set the invoice to closed in the system. If at any time a payment is voided and the balance becomes non-zero, the invoice status will be changed to open.
